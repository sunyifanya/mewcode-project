# spec.md — Slash Command System（斜杠命令注册与分发）

## 背景

当前 TerminalUI 中斜杠命令硬编码在一个 `switch` 语句里（`TerminalUI.java:150-230`）：每个命令直接写死匹配字符串和处理逻辑，无元数据、无别名、无补全、无冲突检测。新增命令需要改核心 UI 代码，也容易写错别名而不自知。

参考项目（mewcode-java）已实现了一套基础的 Command 注册机制，但缺少别名冲突启动检测、命令解析器、Tab 补全、子命令支持等能力。

本 spec 描述将此机制完整迁移到 MewCode 并增强的设计。

## 目标用户

使用 MewCode 的开发者，希望用斜杠开头的输入快速执行本地操作（清屏、查状态、切换模式、压缩上下文等），绕过 LLM 省 Token、响应快、行为确定。

典型场景：
- 输入 `/plan` 进入只读调研模式，状态栏立即显示 `[PLAN]`，不需要 LLM 解释
- 输入 `/help status` 查看某个命令的详细用法
- 输入 `/c`（别名）触发 `/compact`，少打几个字
- 输入 `/` 后按 Tab，弹出可用命令菜单，选择确认

## 能力清单

### A. 命令注册中心

1. **命令元数据登记**：每条命令定义包含名称、别名列表、简短描述、用法示例、命令类型、可选参数提示、是否隐藏，以及处理函数（LOCAL 和 PROMPT 类型需要，LOCAL_UI 类型可无）。名称和别名均为不含斜杠前缀的纯小写字符串。

2. **启动期别名冲突检测**：注册阶段检查新命令的名称及每个别名是否与已注册命令的名称或别名重复（大小写不敏感）。检测到冲突时立即 panic 退出（`System.exit(1)` 并输出冲突详情），不等运行时再炸。

3. **命令查询**：支持按名称或别名的精确匹配查找；支持按前缀模糊搜索（大小写不敏感），只返回可见命令，排序后输出。

4. **可见/隐藏控制**：标记为隐藏的命令不参与 `/help` 列表展示，不参与模糊搜索，不参与 Tab 补全，但精确匹配仍可执行。

### B. 命令解析与分发

5. **斜杠识别与解析**：输入以 `/` 开头即判定为命令。取第一个空格前为命令名（去掉 `/` 前缀，转小写），第一个空格后到行尾为参数字符串。命令名大小写不敏感。

6. **空输入与未命中处理**：`/` 后无内容视为空命令，不执行任何操作。命令名未注册时输出"未知命令"提示，并引导用户输入 `/help` 查看可用命令。

7. **三分支执行模式**：
   - **LOCAL**（纯本地）：执行处理函数，返回文本字符串，直接显示给用户，不经过 LLM
   - **LOCAL_UI**（影响界面状态）：不返回文本，直接操作界面状态（清屏、切换模式、退出等），由 TerminalUI 内部分发
   - **PROMPT**（预设提示词）：执行处理函数生成提示词文本，构造为用户消息注入对话，交给 LLM 处理

8. **用户入口分流器**：在用户回车处理入口处判断输入是否以 `/` 开头——是则走命令分发（本地处理），否则走正常 LLM 对话流程。两个路径互不干扰。

### C. 别名与 Tab 补全

9. **别名支持**：每条命令可注册零个或多个别名。别名和主名称在匹配、搜索时地位平等（大小写不敏感）。别名冲突在注册期即被拦截。

10. **Tab 补全**：用户输入 `/` 及若干字符后按 Tab 键触发补全——模糊搜索非隐藏命令。匹配结果仅一条时直接补全命令名并追加一个尾随空格；匹配结果多条时弹出选择菜单（上下键导航、Enter 或 Tab 确认、Esc 关闭）。隐藏命令不参与补全。

### D. 界面控制抽象

11. **CommandContext 运行时状态注入**：定义命令上下文对象，通过函数式接口（`Supplier`、`IntSupplier` 等）向 LOCAL 和 PROMPT 命令处理器提供运行时数据——参数字符串、工作目录、模型名、权限模式、计划模式状态、工具数量、Token 用量、记忆列表、会话信息等。命令处理器只依赖此上下文接口，不绑定具体渲染框架。

12. **状态栏联动模式标记**：状态栏显示当前工作模式标记——`[PLAN]`（计划模式）或 `[DEFAULT]`（执行模式）。`/plan` 进入计划模式后标记立即切换为 `[PLAN]`；`/do` 退回执行模式后标记恢复 `[DEFAULT]`。`/mode` 循环权限模式不改变此标记。

### E. 内置命令

13. **/help**（LOCAL）—— 显示命令列表或单个命令详情。无参数时列出所有可见命令（名称、别名、描述）；带参数时显示指定命令的详细用法。别名：`h`、`?`。

14. **/status**（LOCAL）—— 显示当前运行状态（工作目录、模型、权限模式、Token 用量、工具数量、记忆条数）。别名：`s`。

15. **/compact**（LOCAL）—— 触发上下文压缩，返回压缩结果文本。别名：`c`。

16. **/session**（LOCAL）—— 会话管理。子命令 `list` 列出所有存档会话，`info` 显示当前会话信息。无子命令时默认等同于 `list`。

17. **/memory**（LOCAL）—— 记忆管理。子命令 `list` 列出所有自动记忆，`clear` 清除全部记忆。无子命令时默认等同于 `list`。

18. **/permission**（LOCAL）—— 权限管理。子命令 `info` 显示当前权限模式，`mode <模式名>` 切换到指定模式。别名：`perm`。

19. **/clear**（LOCAL_UI）—— 清空当前对话历史和界面消息列表，重置计划模式状态。

20. **/plan**（LOCAL_UI）—— 进入计划（只读调研）模式，限制 Agent 仅能使用只读工具。别名：`p`。

21. **/do**（LOCAL_UI）—— 退出计划模式，恢复到进入计划模式前的权限模式。

22. **/mode**（LOCAL_UI）—— 循环切换权限模式（DEFAULT → ACCEPT_EDITS → BYPASS → DEFAULT），跳过 PLAN 模式（PLAN 仅通过 `/plan` 进入）。

23. **/exit**（LOCAL_UI）—— 退出 MewCode。别名：`q`。

24. **/resume**（LOCAL_UI）—— 打开会话恢复选择界面。别名：`r`。

25. **/review**（PROMPT）—— 将"审查当前代码变更"的预设提示词注入对话，交给 LLM 处理。支持可选参数指定额外关注点。

## 非功能要求

- 命令注册与冲突检测在启动阶段完成，典型耗时 <10ms（13 条命令内存操作）
- 命令匹配（精确或模糊搜索）典型耗时 <1ms（13 条命令的线性扫描）
- Tab 补全响应 <100ms（含文件系统操作，如有路径补全需求）
- 所有命令处理不抛未捕获异常——错误以文本形式返回显示
- 命令注册中心的查找操作是线程安全的（会话恢复可能在后台线程触发命令查询）

## 设计骨架

```
com.mewcode.command/                              ← 新包
├── Command                                        ← 命令元数据记录（名称、别名、描述、用法、类型、隐藏、参数提示）
├── CommandType                                    ← 枚举：LOCAL / LOCAL_UI / PROMPT
├── CommandContext                                 ← 运行时上下文（函数式注入，不依赖具体 UI 框架）
└── CommandRegistry                                ← 注册中心（注册、冲突检测、搜索、查找、执行）

修改:
├── tui/TerminalUI.java                            ← 接入 CommandRegistry，替换硬编码 switch，添加分流器 + Tab 补全 + 状态栏简化
└── MewCode.java                                   ← 启动时初始化 CommandRegistry，构建 CommandContext，传入 TerminalUI
```

### 关键数据流

```
MewCode 启动:
  1. new CommandRegistry() → 自动注册 13 条内置命令 + 冲突检测
  2. 将 CommandRegistry 传入 TerminalUI

用户输入处理:
  3. TerminalUI.start() 读入用户输入
     → 以 / 开头？
        是 → 命令 Parser 提取 name + args
             → CommandRegistry.find(name)
                命中 → 按 CommandType 分发：
                  LOCAL     → execute(name, ctx) → 显示返回文本
                  LOCAL_UI  → TerminalUI 内部 switch 处理 → 直接操作界面
                  PROMPT    → execute(name, ctx) → 构造用户消息 → 注入对话 → 交给 LLM
                未命中 → 显示"未知命令"提示 + /help 引导
        否 → 交给 Agent Loop 正常对话

Tab 补全:
  4. 用户输入 /<prefix> 后按 Tab
     → CommandRegistry.search(prefix) → 非隐藏命令匹配
        1 条 → 直接补全命令名 + " "
        N 条 → 弹出选择菜单 → 用户确认 → 补全
        0 条 → 不操作

状态栏:
  5. 每次渲染时根据 planMode 布尔值显示 [DEFAULT] 或 [PLAN]
```

## Out of Scope

- 用户自定义命令（留给后续 Skill 系统）
- 动态生成提示词（PROMPT 命令的提示词在注册时写死）
- 命令级权限控制（如禁止在某种模式下执行某命令——当前所有命令在任何模式下可执行）
- 命令别名国际化
- 命令执行日志/审计
- 命令嵌套管道（一个命令的输出作为另一个命令的输入）
- 斜杠命令参数的类型校验和自动转换（参数始终以纯字符串传递）
